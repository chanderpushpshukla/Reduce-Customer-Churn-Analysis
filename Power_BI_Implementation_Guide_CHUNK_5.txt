SECTION 1: DATA REQUIREMENTS & PREPROCESSING (Q10‚ÄìQ12)
================================================================================

1.1 DATA SOURCES & COLUMNS

For Q10 (Satisfaction):
  Source: fact_customer_churn
  Columns Required:
    - Satisfaction_Score (numeric, 1‚Äì5 scale): CSAT rating
    - NPS_Score (numeric, -100 to +100, optional): Net Promoter Score
    - Complaint_Flag (Yes/No): Whether customer filed complaint
    - Churn_Flag
    - Customer_ID
    - Report_Month
  
  Data Quality:
    - Satisfaction_Score: 1‚Äì5 only, no nulls (fill blanks with median 3)
    - NPS_Score: -100 to +100 (if present); convert to NPS category (Detractor, Passive, Promoter)
    - No outliers (e.g., satisfaction = 10)

For Q11 (Risk Segmentation):
  Source: fact_customer_churn + all dimensions (Age, Contract, Payment, Region, Tenure, Usage)
  Columns Required:
    - ALL dimensions from Q4‚ÄìQ9 (Age_Group, Contract_Type, Region, Tenure_Bucket, etc.)
    - Churn_Risk_Profile (calculated in Chunk 4, Section 3.1)
    - Customer_Segment (if available: Retail, SME, Corporate)
    - Monthly_Charges
    - Total_Revenue
  
  Data Quality:
    - All dimensions populated (no unknowns)
    - Monthly_Charges > 0 (no free tier; or separate free tier analysis)
    - Churn_Risk_Profile calculated and validated

For Q12 (Financial Impact):
  Source: fact_customer_churn
  Columns Required:
    - Churn_Flag
    - Customer_ID
    - Total_Revenue (cumulative revenue from customer to date)
    - Monthly_Charges (recurring monthly fee)
    - Join_Date (to calculate lifetime)
    - Churn_Date (to calculate revenue at loss date)
    - Acquisition_Cost (optional; for LTV calculation)
  
  Data Quality:
    - Total_Revenue: Non-negative, realistic (validate outliers)
    - Monthly_Charges: Consistent; matches contract type pricing
    - No negative revenue (refunds handled separately)
    - Currency: Consistent (USD, EUR, etc.; not mixed)

1.2 POWER QUERY PREPROCESSING

For Q10, standardize Satisfaction_Score:

  STEP 1: Fill Blanks with Median
  - Select Satisfaction_Score
  - Transform > Replace Values
  - Find: (blank/error)
  - Replace: 3 (median of 1‚Äì5 scale)
  - Click Replace All

  STEP 2: Create NPS Category (if NPS_Score available)
  - Add Column > Conditional Column
  - Name: "NPS_Category"
  - Formula:
    if [NPS_Score] >= 9 then "Promoter"
    else if [NPS_Score] >= 7 then "Passive"
    else "Detractor"

  STEP 3: Create Satisfaction Tier
  - Add Column > Conditional Column
  - Name: "Satisfaction_Tier"
  - Formula:
    if [Satisfaction_Score] >= 4 then "Satisfied"
    else if [Satisfaction_Score] = 3 then "Neutral"
    else "Dissatisfied"

For Q11, ensure all dimensions present (from Chunks 3‚Äì4):

  Verify columns exist:
    - Age_Group (from Q4)
    - Contract_Type (from Q5)
    - Payment_Method (from Q6)
    - Tenure_Bucket (from Q7)
    - Usage_Category (from Q8)
    - Region (from Q3)
    - Customer_Segment (if available)
    - Churn_Risk_Profile (calculated in Chunk 4)

For Q12, validate financial columns:

  STEP 1: Ensure Monthly_Charges > 0
  - Filter > Remove rows where Monthly_Charges <= 0
  - (If free tier needed, create separate analysis)

  STEP 2: Calculate Lifetime Value
  - Add Column > Custom Column
  - Name: "Customer_LTV"
  - Formula: [Total_Revenue] (if already cumulative)
    OR: [Monthly_Charges] * [Tenure_Months] (if need to calculate)

  STEP 3: Calculate Months Until Churn (for projections)
  - Add Column > Custom Column
  - Name: "Months_to_Churn"
  - Formula:
    if [Churn_Flag] = TRUE then
      DATEDIFF([Churn_Date], [Join_Date], "M")
    else
      DATEDIFF(TODAY(), [Join_Date], "M") // Still active

2.1 SATISFACTION MEASURES (Q10)

MEASURE 1: Average Satisfaction Score (All Customers)
Purpose: Monitor overall customer satisfaction; leading indicator for retention
Audience: CEO, VP Customer Success, Board (as retention risk metric)

DAX CODE:
  Avg Satisfaction (All) = AVERAGE('fact_customer_churn'[Satisfaction_Score])
  
  Avg Satisfaction Churned = CALCULATE(
    AVERAGE('fact_customer_churn'[Satisfaction_Score]),
    'fact_customer_churn'[Churn_Flag] = TRUE()
  )
  
  Avg Satisfaction Active = CALCULATE(
    AVERAGE('fact_customer_churn'[Satisfaction_Score]),
    'fact_customer_churn'[Churn_Flag] = FALSE()
  )

Comment: Satisfaction 4.0+ = healthy; 3.0‚Äì3.9 = at-risk; <3.0 = critical.
Format: Decimal, 1 place (e.g., 3.7/5.0)

MEASURE 2: Satisfaction Distribution (Count by Tier)
Purpose: Segment customers by satisfaction level for targeted intervention
Audience: Customer Success, Retention teams

DAX CODE:
  Satisfied Customers (4-5) = CALCULATE(
    DISTINCTCOUNT('fact_customer_churn'[Customer_ID]),
    'fact_customer_churn'[Satisfaction_Score] >= 4
  )
  
  Neutral Customers (3) = CALCULATE(
    DISTINCTCOUNT('fact_customer_churn'[Customer_ID]),
    'fact_customer_churn'[Satisfaction_Score] = 3
  )
  
  Dissatisfied Customers (1-2) = CALCULATE(
    DISTINCTCOUNT('fact_customer_churn'[Customer_ID]),
    'fact_customer_churn'[Satisfaction_Score] <= 2
  )

Comment: Used for waterfall chart and segmentation
Format: Whole number (e.g., 850, 600, 250)

MEASURE 3: Satisfaction-Churn Linkage (Cross-tabulation measure)
Purpose: Quantify churn risk by satisfaction level
Audience: Analytics, Product (evidence of satisfaction ‚Üí retention link)

DAX CODE:
  Churn Rate by Satisfaction = 
    VAR SatisfactionGroup = VALUES('fact_customer_churn'[Satisfaction_Score])
    RETURN
      IF(
        HASONEVALUE('fact_customer_churn'[Satisfaction_Score]),
        [Churn Rate %],
        BLANK()
      )

  // Alternative explicit version:
  Churn Rate at Satisfaction Tier = 
    VAR TierCount = DISTINCTCOUNT('fact_customer_churn'[Customer_ID])
    VAR TierChurned = CALCULATE(
      DISTINCTCOUNT('fact_customer_churn'[Customer_ID]),
      'fact_customer_churn'[Churn_Flag] = TRUE()
    )
    RETURN
      IF(TierCount = 0, BLANK(), DIVIDE(TierChurned, TierCount) * 100)

EXPECTED OUTPUT:
  Satisfaction 5: 3% churn
  Satisfaction 4: 5% churn
  Satisfaction 3: 18% churn (cliff)
  Satisfaction 2: 35% churn
  Satisfaction 1: 60% churn

MEASURE 6: Revenue-Weighted Risk Score
Purpose: Identify highest-impact risk segments (account for customer value)
Audience: Finance, Sales, CFO

DAX CODE:
  Revenue at Risk = 
    CALCULATE(
      SUM('fact_customer_churn'[Monthly_Charges]) * 
        COUNTROWS(
          FILTER(
            'fact_customer_churn',
            'fact_customer_churn'[Churn_Risk_Profile] = "Critical"
          )
        ),
      'fact_customer_churn'[Churn_Flag] = FALSE() // Only at-risk, still active
    )

  // Simpler alternative:
  Revenue from Critical Risk = CALCULATE(
    SUM('fact_customer_churn'[Monthly_Charges]),
    'fact_customer_churn'[Churn_Risk_Profile] = "Critical",
    'fact_customer_churn'[Churn_Flag] = FALSE()
  )

Comment: Weights risk by revenue impact. Example: 50 critical customers @ $100/mo each 
         = $5,000/month revenue at risk.
Format: Currency ($), 0 decimals (e.g., $15,000)

2.3 FINANCIAL IMPACT MEASURES (Q12)

MEASURE 7: Total Revenue Lost from Churn
Purpose: Quantify financial impact of churn for business case
Audience: CFO, Board, Finance

DAX CODE:
  Total Revenue Lost (Churned) = CALCULATE(
    SUM('fact_customer_churn'[Total_Revenue]),
    'fact_customer_churn'[Churn_Flag] = TRUE()
  )

Comment: Cumulative revenue lost from all churned customers to date.
Example: 679 churned customers √ó avg $150 revenue = $101,850 loss
Format: Currency ($), 0 decimals

MEASURE 8: Monthly Revenue Churn (Recurring Revenue Lost)
Purpose: Forecast recurring revenue impact (MRR loss)
Audience: Finance, CFO (revenue forecast)

DAX CODE:
  MRR Lost to Churn = CALCULATE(
    SUM('fact_customer_churn'[Monthly_Charges]),
    'fact_customer_churn'[Churn_Flag] = TRUE()
  )

Comment: Annualized MRR loss = MRR Lost √ó 12
Example: If 60 customers churn/month @ avg $50/mo each = $3,000 MRR loss = $36,000 ARR
Format: Currency ($), 0 decimals

MEASURE 9: Customer Lifetime Value (LTV) by Segment
Purpose: Value each customer segment to inform retention investment priorities
Audience: CFO, Finance (payback analysis)

DAX CODE:
  Avg LTV = AVERAGE('fact_customer_churn'[Customer_LTV])
  
  // If Customer_LTV not pre-calculated:
  Avg LTV (Calculated) = 
    AVERAGE(
      'fact_customer_churn'[Monthly_Charges] * 'fact_customer_churn'[Tenure_Months]
    )
  
  // By segment (e.g., by Contract Type):
  LTV by Contract = 
    CALCULATE(
      AVERAGE('fact_customer_churn'[Customer_LTV]),
      ALL('fact_customer_churn'[Churn_Flag]) // Average of all, not filtered by churn
    )

Comment: Example LTVs:
  - Monthly contract: Avg $500 LTV (short tenure)
  - Annual contract: Avg $1,800 LTV (long tenure)
  - Insight: Annual contracts worth 3.6x monthly; justifies 20% discount

Format: Currency ($)

MEASURE 10: Revenue at Risk (Projected Annual Loss)
Purpose: Forecast annual revenue impact if churn continues at current rate
Audience: CFO, Executive team

DAX CODE:
  Annual Revenue at Risk = [MRR Lost to Churn] * 12

  // Alternative: Project current churn rate forward
  Projected Annual Churn Loss (Cohort) = 
    VAR CurrentChurnRate = [Churn Rate %] / 100
    VAR ActiveCustomers = [Active Customers]
    VAR AvgMonthlyCost = AVERAGE('fact_customer_churn'[Monthly_Charges])
    VAR ProjectedMonthlyChurn = ActiveCustomers * CurrentChurnRate
    RETURN
      ProjectedMonthlyChurn * AvgMonthlyCost * 12

Comment: If current churn 15% / 1,000 active customers @ $75/mo average:
  Projected annual loss = 150 churn/month √ó $75 √ó 12 = $135,000 ARR at risk

Format: Currency ($)

MEASURE 11: Payback Period (for retention investment)
Purpose: Justify retention spend (if CS investment = $X, payback = Y months)
Audience: CFO, Finance

DAX CODE:
  Payback Period Months = 
    VAR RetentionInvestment = 50000 // Example: $50K/year CS team cost
    VAR MonthlyRetentionSavings = [MRR Lost to Churn]
    RETURN
      IF(MonthlyRetentionSavings = 0, BLANK(), RetentionInvestment / MonthlyRetentionSavings)

Comment: If $50K investment prevents $10K MRR churn = 5 months payback (ROI > 2x annually)
Example calculation:
  - Current MRR churn: $3,000/month ($36K/year lost)
  - CS team cost: $120K/year
  - Assumed prevention: 30% churn reduction (save $10.8K/year)
  - ROI = $10.8K / $120K = 9% direct ROI (not including:
    improved NPS, reduced acquisition to replace, brand damage prevention)

Format: Decimal, 1 place (e.g., 5.4 months)
